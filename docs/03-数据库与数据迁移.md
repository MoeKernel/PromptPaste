# 数据库与数据迁移

PromptPaste 使用 SQLite 存储片段、分类、标签和迁移版本信息。

## 文件位置

| 类型 | 默认位置 |
|------|----------|
| 默认数据库 | `%USERPROFILE%\.promptpaste\promptpaste.db` |
| 设置文件 | `%USERPROFILE%\.promptpaste\settings.json` |
| 日志目录 | `%USERPROFILE%\.promptpaste\logs` |
| 备份目录 | `%USERPROFILE%\.promptpaste\backups` |

用户也可以通过 `文件 -> 打开数据库...` 打开或创建其他 `.db` 文件。

## 表结构

### `clipboard_items`

存储片段本体。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK | 自增主键 |
| `title` | TEXT NOT NULL | 标题 |
| `content` | TEXT NOT NULL | 正文 |
| `item_type` | TEXT NOT NULL | 兼容旧版的首个分类路径 |
| `usage_count` | INTEGER | 使用次数 |
| `last_used` | TEXT | 最近使用时间，ISO 8601 字符串 |
| `created_at` | TEXT | 创建时间，ISO 8601 字符串 |
| `deleted_at` | TEXT | 进入回收站时间；为空表示正常片段 |

### `categories`

存储多级分类树。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK | 自增主键 |
| `name` | TEXT NOT NULL | 分类名 |
| `parent_id` | INTEGER NULL | 父分类 ID，空表示根分类 |
| `sort_order` | INTEGER | 排序字段，当前主要按名称展示 |
| `created_at` | TEXT | 创建时间 |

唯一索引：`IFNULL(parent_id, 0), name`，确保同一父级下分类名唯一。

### `item_categories`

片段与分类的多对多关系。

| 字段 | 类型 | 说明 |
|------|------|------|
| `item_id` | INTEGER | 片段 ID |
| `category_id` | INTEGER | 分类 ID |

主键：`item_id, category_id`。

### `tags` 与 `item_tags`

`tags` 存储标签字典，`item_tags` 存储片段与标签的多对多关系。

### `schema_info`

存储数据库结构版本。

| key | value |
|-----|-------|
| `schema_version` | 当前版本号，当前代码定义为 `2` |

## 迁移策略

- 应用打开数据库时会执行 `CreateTables()`。
- 如旧库缺少 `deleted_at` 字段，会先备份数据库，再执行 `ALTER TABLE`。
- 旧版 `item_type` 会迁移为分类路径，并写入 `item_categories`。
- 新表使用 `CREATE TABLE IF NOT EXISTS`，可重复执行。

## 回收站数据规则

- 正常列表只读取 `deleted_at IS NULL` 的片段。
- 回收站只读取 `deleted_at IS NOT NULL` 的片段。
- 移入回收站只更新 `deleted_at`，不会删除分类/标签关系。
- 恢复回收站只清空 `deleted_at`。
- 永久删除会删除片段本体和对应分类/标签关系。
- 清空回收站只物理删除已标记删除的片段。

## 分类合并规则

- 同一父级下存在同名分类时，拖拽移动会触发合并。
- 合并时会把源分类的片段关系迁移到目标分类。
- 子分类若与目标分类下子分类同名，则递归合并。
- 合并完成后删除源分类。

## 备份触发点

当前实现会在以下操作前尝试备份当前数据库：

- 数据库结构迁移。
- JSON 导入。
- 删除分类。
- 移入回收站。
- 永久删除。
- 清空回收站。
- 选项窗口中手动点击“立即备份数据库”。

备份失败不会阻止主流程，但会写入日志。
