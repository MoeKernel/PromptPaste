# JSON 导入导出格式

PromptPaste 使用 JSON 数组导入和导出片段。该格式对应 `ExportClipboardItem` 模型。

## 示例

```json
[
  {
    "title": "AI提示词示例",
    "content": "你是一名专业的{{领域}}专家，请帮我{{任务}}。",
    "categoryPaths": ["AI提示词/写作", "常用"],
    "tags": ["prompt", "写作"],
    "usageCount": 0,
    "lastUsed": null,
    "createdAt": "2026-06-11T00:00:00+08:00"
  },
  {
    "title": "商务邮件模板",
    "content": "尊敬的{{收件人}}：\n\n感谢您的来信。关于{{问题}}，我们会尽快处理。",
    "categoryPaths": ["商务话术/邮件"],
    "tags": ["email", "商务"],
    "usageCount": 0,
    "lastUsed": null,
    "createdAt": "2026-06-11T00:00:00+08:00"
  }
]
```

## 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` | string | 是 | 片段标题，空白会跳过该记录 |
| `content` | string | 是 | 片段正文，空白会跳过该记录 |
| `categoryPaths` | string[] | 否 | 分类路径，使用 `/` 表示多级分类 |
| `tags` | string[] | 否 | 标签列表 |
| `usageCount` | number | 否 | 使用次数；导入时小于 0 会按 0 处理 |
| `lastUsed` | string/null | 否 | 最近使用时间，建议 ISO 8601 |
| `createdAt` | string | 否 | 创建时间；缺失或默认值时按当前时间处理 |

导入反序列化使用大小写不敏感配置，因此 `CategoryPaths` / `categoryPaths` 这类大小写差异可以兼容。

## 分类路径规则

- `"AI提示词/写作"` 会创建或复用根分类 `AI提示词`，再创建或复用子分类 `写作`。
- 一个片段可以指定多个分类路径。
- 如果未提供分类路径或路径为空，导入时会归入 `未分类`。

## 标签规则

- 标签会去除首尾空白。
- 空标签会被忽略。
- 同一片段内标签按大小写不敏感去重。

## 导入容错规则

导入时：

- JSON 必须是数组。
- 标题或正文为空的记录会跳过。
- 缺失分类时自动使用 `未分类`。
- 缺失标签时按空标签列表处理。
- 缺失创建时间时使用当前时间。
- 导入前会提示确认并备份当前数据库。

## 导出规则

导出时：

- 只导出未进入回收站的片段。
- 导出分类路径而不是分类 ID，便于跨数据库导入。
- 导出标签名称而不是标签 ID。
- 日期以 JSON 序列化器输出的日期格式保存。

## UI 入口

- 导入：`文件 -> 导入数据`
- 导出：`文件 -> 导出数据`
- 格式说明：`帮助 -> JSON格式说明`
