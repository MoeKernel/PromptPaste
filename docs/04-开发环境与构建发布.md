# 开发环境与构建发布

## 环境要求

- Windows 10/11。
- .NET SDK 8.x。
- PowerShell 或 `pwsh.exe`。
- WPF 运行环境随 .NET 桌面运行时提供。

## 项目入口

| 文件 | 说明 |
|------|------|
| `PromptPaste.sln` | 解决方案文件 |
| `PromptPaste/PromptPaste.csproj` | C# WPF 项目文件 |
| `PromptPaste/App.xaml.cs` | 应用入口、托盘和生命周期 |
| `PromptPaste/MainWindow.xaml` | 主窗口 UI |
| `PromptPaste/MainWindow.xaml.cs` | 主窗口交互逻辑 |

## 常用开发命令

在仓库根目录执行：

```bash
rm -rf -- PromptPaste/obj PromptPaste/bin
project_win=$(wslpath -w PromptPaste/PromptPaste.csproj)
/mnt/c/Windows/System32/WindowsPowerShell/v1.0/powershell.exe -NoProfile -ExecutionPolicy Bypass -Command "dotnet build \"$project_win\" -c Release"
```

启动冒烟检查：

```bash
exe_win=$(wslpath -w PromptPaste/bin/Release/net8.0-windows/PromptPaste.exe)
/mnt/c/Windows/System32/WindowsPowerShell/v1.0/powershell.exe -NoProfile -ExecutionPolicy Bypass -Command '$p = Start-Process -FilePath "'"$exe_win"'" -PassThru; Start-Sleep -Seconds 3; if ($p.HasExited) { Write-Host "EXITED:$($p.ExitCode)" } else { Write-Host "RUNNING"; Stop-Process -Id $p.Id -Force }'
```

## 自动化测试

测试项目位于 `PromptPaste.Tests`，当前覆盖文本处理、数据库 CRUD、分类/标签、回收站、分类合并、JSON 导入导出、快速候选搜索、快速候选位置计算、DPI 坐标转换和剪贴板内部复制过滤。

```bash
sln_win=$(wslpath -w PromptPaste.sln)
/mnt/c/Windows/System32/WindowsPowerShell/v1.0/powershell.exe -NoProfile -ExecutionPolicy Bypass -Command "dotnet test "$sln_win" -c Release"
```

也可以按测试类过滤：

```bash
/mnt/c/Windows/System32/WindowsPowerShell/v1.0/powershell.exe -NoProfile -ExecutionPolicy Bypass -Command "dotnet test PromptPaste.Tests/PromptPaste.Tests.csproj -c Release --filter TextProcessorTests"
```

## 发布脚本

根目录 `build.ps1` 用于发布正式包。

```powershell
pwsh.exe -NoProfile -ExecutionPolicy Bypass -File .\build.ps1
```

如果没有安装 PowerShell 7，也可以使用 Windows PowerShell：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\build.ps1
```

脚本行为：

- 默认 `Release` 构建。
- 使用 `dotnet publish` 输出到临时目录。
- 最终发布包放在仓库根目录 `bin`。
- 构建中间目录使用 `.build` 和项目 `obj`，结束后清理。
- 不生成调试符号；发布后删除 `.pdb` 和 `.xml`。
- 成功发布后复制到 `D:\Protable\PromptPaste`，目录不存在时创建。
- 构建成功后更新 `docs/log/build-success-count.txt`，记录成功次数、最近成功时间、配置和输出目录；该文件已被 `.gitignore` 忽略。
- 如果部署目录中文件被正在运行的程序占用，会给出 warning，构建结果仍保留在根目录 `bin`。

指定配置：

```powershell
pwsh.exe -NoProfile -ExecutionPolicy Bypass -File .\build.ps1 -Configuration Release
```

## 发布包说明

根目录 `bin` 是可分发目录，通常包含：

- `PromptPaste.exe`
- `PromptPaste.dll`
- `.deps.json` / `.runtimeconfig.json`
- SQLite 和托盘依赖 DLL
- `runtimes/`：NuGet native runtime 依赖，SQLite 原生库需要保留
- `resources/icons/app_icon.ico`

不应包含：

- `.pdb`
- `.xml`
- `obj`
- `.codegraph`
- Python 残留：`src`、`main.py`、`requirements.txt`、`resources/styles`

## 图标资源

- 应用图标源文件：`resources/icons/app_icon.svg`
- Windows 可执行文件图标：`resources/icons/app_icon.ico`
- 项目文件通过 `ApplicationIcon` 指向 `.ico`，并把 `.svg` 作为资源链接进项目。

## 开发注意事项

- 修改数据库结构时必须考虑旧数据库迁移和备份。
- 修改剪贴板逻辑时必须保留内部复制过滤，避免复制卡片内容、变量替换后的内容或恢复原剪贴板内容后又提示保存。
- 修改热键唤出逻辑时需验证 Windows 10/11 虚拟桌面行为。
- 修改快速候选定位时需验证单显示器 DPI、多显示器混合 DPI、屏幕右侧和底部边界。
- 修改 UI 样式时需验证顶栏输入框、卡片固定高度、日期显示和按钮 hover 状态。
